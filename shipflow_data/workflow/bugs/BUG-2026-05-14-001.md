# Bug Record: BUG-2026-05-14-001

## Summary

- Title: Dependabot uses old Tubeflow directories after ReplayGlowz rename
- Current status: `fixed-pending-verify`
- Severity: `medium`
- Owner: Diane
- Next command: `/sf-test --retest BUG-2026-05-14-001`

## Reproduction

- Preconditions:
  - GitHub Dependabot runs against `diane-defores/replayglowz` on `main`.
  - `.github/dependabot.yml` still references old `tubeflow_*` directories.
- Steps:
  1. Trigger Dependabot updates for npm, pub, pip, or docker ecosystems.
  2. Dependabot clones the repo and tries to fetch dependency files.
  3. Dependabot looks for `/tubeflow_site`, `/tubeflow_app`, or `/tubeflow_lab`.

## Expected Behavior

- Dependabot should scan the current ReplayGlowz directories:
  - `/replayglowz_site`
  - `/replayglowz_app`
  - `/replayglowz_lab`

## Observed Behavior

- Dependabot jobs fail with `dependency_file_not_found` because the configured directories no longer exist after the ReplayGlowz rename.
- The Node.js 20 deprecation annotation is a GitHub-hosted `github/dependabot-action@main` warning, not the root cause of these failed update jobs.

## Evidence

- `gh run view 25888204759 --log`: `/tubeflow_app not found`
- `gh run view 25888204217 --log`: `/tubeflow_site not found`
- `gh run view 25888203677 --log`: `/tubeflow_lab not found`
- `.github/dependabot.yml` before fix referenced the old paths.
